# CODEX - Bitácora Terreno

Carpeta de trabajo para modificar la Bitácora sin tocar directamente el archivo original.

## Archivo De Trabajo

- `bitacora-terreno-v3_9.html`

## Reglas

- Trabajar un cambio específico a la vez.
- Crear backup antes de cada cambio relevante.
- Mantener máximo 3 backups recientes por herramienta.
- No subir a GitHub ni reemplazar el original hasta que Ignacio valide.
- Mantener la coherencia visual y funcional de la herramienta existente.
- Evitar refactors grandes si no son necesarios para el cambio solicitado.

## Flujo

1. Codex modifica el archivo dentro de esta carpeta.
2. Ignacio prueba el archivo local.
3. Si queda bien, se versiona.
4. Solo al final se decide si reemplaza el original o se sube a GitHub.
